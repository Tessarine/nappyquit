# Instructions

This skill describes how to write AGENTS.md files in subdirectories.

## Directory Discovery

When creating an AGENTS.md file in a subdirectory:
1. First explore the directory structure to understand the context
2. Look for similar AGENTS.md files in parent or sibling directories for reference
3. Analyze the content of existing files to identify patterns
4. Condense the patterns found into a 10-30 line description for coding agents

## Content Analysis

When analyzing existing AGENTS.md files:
1. Identify recurring sections (project description, directory structure, principles, etc.)
2. Note the formatting conventions used (markdown syntax, emphasis, code blocks)
3. Extract domain-specific information that should be preserved
4. Determine what information is generic vs. project-specific

## Pattern Condensation

Create a concise description that includes:
1. The purpose of the directory/module
2. Key responsibilities and behaviors
3. Important interfaces or APIs
4. Notable dependencies or relationships
5. Essential coding principles that apply

## Essential Sections

Include these standard sections when appropriate:
- Project/Module description
- Directory structure overview
- Code quality principles
- Localization guidelines (if applicable)
- Testing instructions
- Commit message directives (if needed)

## Best Practices

- Keep descriptions clear and concise (10-30 lines)
- Use markdown formatting consistently
- Include specific examples when helpful
- Reference existing patterns from the codebase
- Focus on what agents need to know to work effectively in this directory
